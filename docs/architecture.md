# Архитектурные особенности реализации сайта

## Общая архитектура

ZeroDolg - это юридический сайт, реализованный с использованием Astro фреймворка с гибридной архитектурой (SSR + SSG). Проект использует островную архитектуру для реализации интерактивных элементов с помощью Preact.

## Слои архитектуры (Feature-Sliced Design)

### app/
Слой приложения содержит:
- Макеты (layouts) - основные структуры страниц
- Стили (styles) - основные стили и настройки Tailwind

### shared/
Общий слой с ресурсами, используемыми по всему приложению:
- analytics - утилиты для интеграции с Google Analytics и Yandex Metrics
- config - конфигурационные файлы и константы
- lib - общие библиотеки и хелперы
- types - глобальные TypeScript типы
- utils - общие утилиты
- api - клиентские API утилиты

### entities/
Слой бизнес-сущностей:
- review - сущность отзывов клиентов
- team - сущность команды

### features/
Слой фич (функциональных возможностей):
- calculator - интерактивный калькулятор банкротства
- forms - обработка форм и интеграция с Bitrix24
- analytics - аналитика и отслеживание событий
- modals - модальные окна

### widgets/
Комплексные компоненты, состоящие из нескольких сущностей:
- header - шапка сайта
- footer - подвал сайта
- faq - блок вопросов-ответов
- reviews - виджет отзывов

### pages/
Слой страниц и API маршрутов:
- api/form - обработка форм и интеграция с Bitrix24
- остальные страницы (index, услуги и т.д.)

## Островная архитектура (Islands Architecture)

Интерактивные компоненты реализованы в директории `islands/` с использованием Preact:
- forms - интерактивные формы
- interactive - интерактивные компоненты (калькулятор, виджеты)
- shared - общие интерактивные компоненты

Эта архитектура позволяет:
- Минимизировать клиентский JavaScript
- Повысить производительность за счет раздельной загрузки интерактивных компонентов
- Сохранить преимущества статической генерации для неинтерактивных элементов

## Интеграция с Bitrix24

Интеграция реализована через:
- API endpoint `/api/form` для обработки форм
- Клиентский компонент `BitrixCallback.astro` с библиотекой `bitrix-callback-widget.ts`
- Валидация данных до отправки
- Обработка ошибок и таймауты

Поддерживаются следующие типы форм:
- callback - заявка на обратный звонок
- consultation - заявка на консультацию
- bankruptcy - заявка на банкротство
- calculator - результаты с калькулятора

## Безопасность и кэширование

### Middleware
Файл `middleware.ts` реализует:
- Заголовки безопасности (CSP, X-Frame-Options, X-Content-Type-Options и др.)
- Кэширование с различными политиками для разных типов ресурсов
- Защиту от CSRF и других уязвимостей

### Заголовки безопасности
- Content Security Policy для предотвращения XSS
- X-Frame-Options для защиты от clickjacking
- Referrer-Policy для контроля referrer заголовка
- Permissions-Policy для ограничения браузерных API

## Производительность

### SSR (Server-Side Rendering)
- Улучшенное SEO за счет рендеринга на сервере
- Быстрая первоначальная загрузка контента
- Поддержка JavaScript-отключенных пользователей

### Оптимизация изображений
- Поддержка WebP и AVIF форматов
- Ленивая загрузка (lazy loading)
- Адаптивные размеры изображений

### CSS оптимизация
- Tree-shaking неиспользуемого CSS
- Custom properties для темизации
- ITCSS архитектура для организации CSS

## Аналитика и отслеживание

- Google Analytics 4 для отслеживания пользовательского поведения
- Yandex Metrics для аудита российских пользователей
- Отслеживание конверсий и пользовательских событий
- Расчет стоимости лида на основе типа формы

## Структура компонентов

Компоненты организованы по функциональности:
- components/ - переиспользуемые UI компоненты
- islands/ - интерактивные компоненты на Preact
- widgets/ - комплексные виджеты из нескольких сущностей
- features/ - функциональные фичи приложения

## Доступность (Accessibility)

- Соответствие WCAG 2.1 AA
- Правильная семантика HTML
- ARIA-атрибуты для интерактивных элементов
- Контрастность цветов по стандартам
- Управление с клавиатуры
