# Архитектура проекта

## Общая архитектура

ZeroDolg Astro использует современную архитектуру с учетом лучших практик веб-разработки:

- **Статическая генерация**: Используется Astro для предварительной генерации HTML
- **Islands Architecture**: Интерактивные компоненты только там, где они нужны
- **Прогрессивное улучшение**: Сайт работает без JavaScript
- **Компонентный подход**: Модульная и переиспользуемая архитектура

## Технологии

### Основные технологии
- **Astro v5.13.5**: Статический генератор сайтов
- **Preact v10.27.1**: Легковесная альтернатива React для интерактивных компонентов
- **TypeScript v5.9.2**: Строгая типизация для JavaScript
- **CSS**: Современный CSS с пользовательскими свойствами

### Архитектурные принципы
- **ITCSS**: Инвертированный треугольник CSS для организации стилей
- **BEM**: Блок, Элемент, Модификатор для именования CSS классов
- **Mobile First**: Дизайн сначала для мобильных устройств

## Компоненты

### Типы компонентов

1. **Статические компоненты** (`/src/components/ui/`, `/src/components/sections/`)
   - Не содержат интерактивности
   - Используются для структурирования контента
   - Полностью рендерятся на сервере

2. **Интерактивные компоненты** (`/src/components/islands/`)
   - Содержат интерактивность через Preact
   - Загружаются только при необходимости
   - Используют Islands Architecture

3. **Макеты** (`/src/layouts/`)
   - Определяют общую структуру страниц
   - Содержат общие элементы (шапка, подвал)

### Архитектура компонентов

#### Структура компонента
```astro
---
// Логика компонента (TypeScript/JavaScript)
---

<!-- HTML разметка -->
<div class="component-name">
  <!-- Семантическая разметка -->
</div>

<!-- CSS стили (опционально) -->
<style>
  /* Стили компонента */
</style>

<!-- JavaScript (опционально) -->
<script>
  /* Интерактивность */
</script>
```

## Стили

### Архитектура ITCSS

1. **Settings**: Переменные и конфигурации
2. **Generic**: Сбросы и нормализация
3. **Elements**: Стили HTML элементов
4. **Components**: Стили UI компонентов
5. **Sections**: Стили секций страниц
6. **Utilities**: Вспомогательные классы

### Методология BEM

Для именования CSS классов используется методология BEM:

```css
/* Блок */
.component-name { }

/* Элемент */
.component-name__element { }

/* Модификатор */
.component-name--modifier { }
.component-name__element--modifier { }
```

## Данные

### Источники данных

1. **Локальный контент** (`/src/content/`)
   - Хранится в формате Markdown и JSON
   - Используется как fallback при недоступности CMS
   - Автоматически обрабатывается Astro Content Collections

2. **Alibaba CMS**
   - Основной источник динамического контента
   - Интеграция через API
   - Сервисы в `/src/lib/`

### Структура данных

#### Коллекции контента
- **team**: Информация о членах команды
- **reviews**: Отзывы клиентов
- **blog**: Блог посты

#### Статические данные
- **calculator.json**: Данные для калькулятора
- **site.json**: Общие данные сайта

## API и сервисы

### Внутренние сервисы

1. **Content Service** (`/src/lib/content-service.js`)
   - Единая точка доступа к контенту
   - Обработка fallback данных
   - Кэширование (при необходимости)

2. **Alibaba CMS API** (`/src/lib/alibaba-cms.js`)
   - Низкоуровневое взаимодействие с CMS
   - Обработка ошибок
   - Логирование

### Внешние API
- **Alibaba CMS**: Основной источник контента
- **Google Analytics**: Аналитика (при настройке)
- **Другие API**: При необходимости

## Производительность

### Оптимизации

1. **Статическая генерация**: Предварительный рендеринг HTML
2. **Ленивая загрузка**: Изображения и компоненты загружаются по мере необходимости
3. **Минификация**: CSS и JavaScript минифицируются при сборке
4. **Оптимизация изображений**: Использование WebP и других современных форматов

### Лучшие практики

1. **Критические CSS**: Важные стили встраиваются в HTML
2. **Асинхронная загрузка**: Скрипты не блокируют рендеринг
3. **Кэширование**: Использование кэширования на уровне CDN
4. **Оптимизация шрифтов**: Предзагрузка критических шрифтов

## Доступность

### ARIA атрибуты
- Все интерактивные элементы имеют соответствующие ARIA атрибуты
- Структура страницы использует семантические HTML элементы
- Навигация доступна с клавиатуры

### Контрастность
- Все тексты соответствуют требованиям контрастности WCAG
- Цветовая схема проверена на доступность

## Безопасность

### Заголовки контента
- Использование CSP для защиты от XSS
- Настройка заголовков безопасности

### Обработка данных
- Валидация всех входных данных
- Экранирование выводимого контента
- Защита от CSRF атак

## Тестирование

### Типы тестов

1. **Unit тесты**: Для утилит и функций
2. **Интеграционные тесты**: Для компонентов
3. **E2E тесты**: Для критических пользовательских сценариев

### Инструменты тестирования
- **Playwright**: Для E2E тестов
- **Jest**: Для unit тестов
- **Lighthouse**: Для аудита производительности и доступности

## Масштабируемость

### Горизонтальное масштабирование
- Статическая генерация позволяет легко масштабировать
- Использование CDN для доставки контента
- Кэширование на всех уровнях

### Вертикальное масштабирование
- Модульная архитектура позволяет легко добавлять функциональность
- Четкое разделение ответственности между компонентами
- Переиспользуемые утилиты и сервисы
