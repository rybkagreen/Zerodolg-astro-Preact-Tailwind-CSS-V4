# Архитектура проекта (Feature-Sliced Design)

## Обзор

ZeroDolg — это корпоративный веб-сайт юридической компании по банкротству физических лиц, созданный с использованием Astro v5.13.5. Проект следует подходу Feature-Sliced Design для организации кодовой базы по бизнес-функциям и слоям, а также архитектуре островов (Islands Architecture) для эффективного сочетания статического контента с интерактивными компонентами.

## Feature-Sliced Design

### Слои архитектуры

#### 1. `app` слой (src/)

- **Назначение**: Входная точка приложения
- **Содержит**: Конфигурационные файлы, глобальные стили, макеты страниц
- **Примеры**: `App.astro`, глобальные стили, шаблоны страниц

#### 2. `processes` слой (src/features/\*/process/)

- **Назначение**: Бизнес-процессы, которые объединяют несколько возможностей
- **Содержит**: Сложные сценарии, которые включают несколько функций
- **Примеры**: Процесс отправки формы заявки, процесс оформления консультации

#### 3. `features` слой (src/features/)

- **Назначение**: Бизнес-функции, которые предоставляют ценность пользователю
- **Содержит**: Логику взаимодействия с UI, состояния, действия
- **Примеры**: формы, модальные окна, аналитика, калькуляторы

#### 4. `entities` слой (src/entities/)

- **Назначение**: Бизнес-сущности предметной области
- **Содержит**: Модели данных, сервисы работы с данными, коллекции
- **Примеры**: отзыв, команда, услуга, статья

#### 5. `shared` слой (src/shared/, src/lib/)

- **Назначение**: Общие утилиты, типы, константы, стили
- **Содержит**: Утилиты, типы TypeScript, константы, хелперы, общие компоненты UI
- **Примеры**: HTTP-клиент, форматирование дат, валидация

### Структура папок по Feature-Sliced Design

```
src/
├── components/             # Переиспользуемые UI компоненты
│   ├── ui/                 # Базовые UI компоненты (Button, Card, etc.)
│   ├── forms/              # Компоненты форм
│   ├── layout/             # Компоненты макета (Header, Footer, etc.)
│   └── sections/           # Секции страниц (Hero, Benefits, etc.)
├── islands/                # Интерактивные Preact компоненты
│   ├── forms/              # Интерактивные компоненты форм
│   ├── interactive/        # Интерактивные компоненты (TeamInteractive, etc.)
│   ├── layout/             # Интерактивные компоненты макета
│   ├── shared/             # Общие интерактивные компоненты
│   └── utils/              # Интерактивные утилиты
├── features/               # Бизнес-функции
│   ├── analytics/          # Функция аналитики
│   ├── calculator/         # Функция калькулятора
│   ├── forms/              # Функция обработки форм
│   ├── modals/             # Функция модальных окон
│   └── [feature-name]/     # Другие бизнес-функции
├── entities/               # Бизнес-сущности
│   ├── review/             # Сущность отзывов
│   ├── team/               # Сущность команды
│   ├── service/            # Сущность услуги
│   └── [entity-name]/      # Другие бизнес-сущности
├── widgets/                # Комплексные UI компоненты, содержащие несколько сущностей
│   ├── hero/               # Виджет героя (главная секция)
│   ├── stats/              # Виджет статистики
│   └── [widget-name]/      # Другие виджеты
├── pages/                  # Страницы сайта
├── layouts/                # Макеты страниц
├── shared/                 # Общие утилиты и функции
├── lib/                    # Вспомогательные функции и сервисы
├── core/                   # Ядро приложения (глобальная логика)
└── styles/                 # Стили (архитектура ITCSS)
```

## Основные принципы

### 1. Статическая генерация

- Сайт генерируется статически при сборке для максимальной производительности
- HTML рендерится на стороне сервера для лучшей SEO оптимизации
- Контент загружается быстрее благодаря предварительной подготовке

### 2. Islands Architecture

- Интерактивные компоненты (острова) изолированы от статического контента
- Используется Preact v10.27.1 для интерактивных элементов
- Интерактивность подключается только по необходимости
- Уменьшается объем JavaScript, необходимого для страницы

### 3. Прогрессивное улучшение

- Сайт полностью функционален без JavaScript
- Интерактивные компоненты добавляются после загрузки страницы
- Обеспечивается доступность для всех пользователей

### 4. Компонентный подход

- Модульные и повторно используемые компоненты
- Четкое разделение ответственности между компонентами
- Простота в поддержке и расширении

## Технологии

- **[Astro](https://astro.build/)** - Статический генератор сайтов (v5.13.5)
- **[Preact](https://preactjs.com/)** - Легковесная альтернатива React (v10.27.1)
- **[TypeScript](https://www.typescriptlang.org/)** - Строгая типизация JavaScript (v5.9.2)
- **[CSS](https://developer.mozilla.org/ru/docs/Web/CSS)** - Современный CSS с пользовательскими свойствами
- **[Tailwind CSS](https://tailwindcss.com/)** - Утилитарный CSS-фреймворк (v4)
- **[Alibaba CMS](https://www.alibabacloud.com/product/cms)** - Облачная система управления контентом

## Стили

- **[ITCSS](https://www.xfive.co/blog/itcss-scalable-maintainable-css-architecture/)** - Инвертированный треугольник CSS для организации стилей
- **[BEM](http://getbem.com/)** - Блок, Элемент, Модификатор для именования CSS классов
- **[Tailwind CSS v4](https://tailwindcss.com/)** - Утилитарный CSS-фреймворк для быстрой разработки
- **Mobile First** - Дизайн сначала для мобильных устройств

## Интеграции

### Alibaba CMS

- Облачная система управления контентом для динамического контента
- Поддержка мультиканального публишинга через Model Context Protocol (MCP)
