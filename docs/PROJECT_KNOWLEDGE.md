## Key learnings & principles

**4 архитектурных принципа (hard constraint во всех промтах):**

1. Правильно сразу — не переделывать.
2. Архитектурно чисто (SOLID, SoC, без патчей поверх патчей); **не нарушать границы FSD-слоёв**.
3. Не хардкодим — тексты/параметры/цены/URL/ключи в config / content / env. Репозиторий
   **публичный** → секреты только в env, `.env` никогда не коммитим.
4. Видим лучше → делаем; quick-and-dirty отклоняем, defer как BL.

Накопленные уроки (этот проект):

- Сайт **в проде** — любое изменение с повышенной осторожностью, gate-checks обязательны.
- Аудит выявил **~12 форм/модалок** — самый тяжёлый по интерактиву блок; новые формы делать по
  образцу (унифицированная механика), не плодить с нуля.
- **Не добавлять** фейковые триггеры (счётчик «онлайн», таймеры «Акция завершена»), keyword-stuffing
  — это активный минус для доверия и SEO (см. дизайн-бриф).
- PII в формах (имя/телефон/сумма долга), вебхук Bitrix24, GA4/Метрика, ключи reCAPTCHA —
  **security-чувствительно**, узкими отдельными промтами.
- Дизайн-система — на **Tailwind + токенах** (config), не на «магических» классах.
- **Методичка ≠ код автоматически** — при первом же housekeeping-проходе (14.07.2026) нашлось
  несколько расхождений (тестовый раннер, CI-триггеры на `master`). Проверять технические
  утверждения методичек против фактического репозитория, не доверять на слово даже собственным
  ранее сгенерированным доками.

## Process rules

- **Git:** feature → `master` через PR (preserve history; без force-push/rebase/squash на
  опубликованных ветках). `master` = production. Секреты/локальная память агента/временное — не в
  репозиторий.
- **CI:** GitHub Actions в репозитории есть, но триггеры не совпадают с `master` (BL-003) — не
  полагаться на них. Обязателен локально: `npm run type-check && npm run lint && npm run build`
  (`npm run test` не входит — раннера нет).
- **Hook discipline:** stop-hook bound at 2 non-trivial acks (идентичные fires = 1 ack) — если
  executor = Claude Code/Fable.
- **Fresh session per рабочий блок** (крупный поток/этап редизайна).
- **PROTECTED:** прод-`master`; живые формы и интеграция Bitrix24; SEO/аналитика/CSP
  (`CLAUDE.md` → «Working conventions» для точного списка путей).
