# API документация для интеграции с Bitrix24

## Общая информация

Интеграция с Bitrix24 осуществляется через REST API и вебхуки. Все запросы требуют аутентификации с использованием OAuth токена.

## Аутентификация

Токены доступа хранятся в переменных окружения:
- `BITRIX24_ACCESS_TOKEN` - токен доступа
- `BITRIX24_REFRESH_TOKEN` - токен обновления
- `BITRIX24_DOMAIN` - домен портала

## Основные операции

### Обработка форм (API Endpoint: /api/form)

Сайт отправляет все формы на `/api/form`, где они обрабатываются и передаются в Bitrix24:

Типы форм:
- `callback` - Заявка на обратный звонок
- `consultation` - Заявка на консультацию
- `bankruptcy` - Заявка на банкротство
- `calculator` - Результаты с калькулятора

Пример обработки данных:
```typescript
// src/pages/api/form.ts
const response = await fetch(`${BITRIX24_DOMAIN}/crm.lead.add.json`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    auth: BITRIX24_ACCESS_TOKEN,
    fields: {
      TITLE: `${formData.type} заявка с сайта`,
      NAME: formData.name,
      PHONE: [{ VALUE: formData.phone, VALUE_TYPE: 'MOBILE' }],
      ...(formData.email && { EMAIL: [{ VALUE: formData.email, VALUE_TYPE: 'WORK' }] }),
      ...(formData.message && { COMMENTS: formData.message }),
    },
  }),
});
```

### Валидация перед отправкой

Все данные проходят клиентскую и серверную валидацию:

1. Проверка формата телефона
2. Проверка валидности email
3. Ограничение длины текста комментариев
4. Проверка обязательных полей

### Обработка ошибок

Компоненты обработки форм включают:
- Таймауты для предотвращения зависания
- Обработку ответов от сервера
- Поддержка тестового режима
- Логирование ошибок

## Вебхуки

### Уведомление о новом лидe

URL: `/api/webhook/bitrix24`

Триггер: `ONCRMLEADADD`

Обрабатываемые поля:
- `NAME` - Имя клиента
- `PHONE` - Телефон клиента
- `EMAIL` - Email клиента
- `COMMENTS` - Комментарии клиента
- `UF_*` - Пользовательские поля

## Калькулятор интеграция

Калькулятор (в `src/features/calculator/`) отправляет результаты в Bitrix24:
- Сумма долга
- Количество кредиторов
- Тип собственности
- Рекомендуемый способ решения

## Обработка ошибок

Все API-запросы возвращают структурированные ответы:

```json
{
  "success": true,
  "result": {},
  "errors": []
}
```

Типичные ошибки:
- `TOKEN_EXPIRED`: Требуется обновить токен
- `INSUFFICIENT_PERMISSIONS`: Недостаточно прав
- `VALIDATION_ERROR`: Ошибка валидации данных
- `TIMEOUT_ERROR`: Превышено время ожидания запроса

## Безопасность

- Все токены должны передаваться через HTTPS
- Валидация всех входящих данных
- Ограничение частоты запросов (rate limiting)
- Ведение логов всех операций
- Поддержка тестового режима для предотвращения лишних запросов на продакшн

## Особенности реализации

### Клиентская библиотека

Компонент `BitrixCallback.astro` и библиотека `bitrix-callback-widget.ts`:
- Маскирование телефонных номеров
- Валидация полей формы
- Обработка ошибок и статусов
- Поддержка нескольких типов форм

### Таймауты и устойчивость

Все запросы к Bitrix24 включают таймауты для предотвращения зависания:
- Таймаут для API запросов: 10 секунд
- Таймаут для webhook: 5 секунд
- Повторные попытки в случае сбоя

## Тестирование

Для тестирования интеграции используйте тестовый портал Bitrix24 с отдельными учетными данными и включите тестовый режим в переменных окружения.
