## 🛠 Инструменты для диагностики

### Debug Helpers

В режиме разработки доступны следующие функции отладки:

```javascript
// Общий обзор всех интерактивных элементов
ZeroDolgDebug.info();

// Проверка элементов FAQ
ZeroDolgDebug.checkFAQ();

// Проверка состояния баннера
ZeroDolgDebug.checkBanner();

// Проверка модальных окон
ZeroDolgDebug.checkModals();

// Тест прокрутки к элементу
ZeroDolgDebug.testScroll('faq');

// Сброс состояния баннера
ZeroDolgDebug.resetBanner();
```

### Логирование

Все компоненты теперь выводят подробные логи в консоль:

- `[Fallback]` - fallback-interactions.js
- `[SpecialOfferBanner]` - баннер акции
- `[ModalManager]` - менеджер модальных окон

## 🚀 Как подключить debug helpers

Добавьте в ваш layout перед закрывающим тегом `</body>`:

```html
<!-- Только для разработки -->
{import.meta.env.DEV && (
<script src="/js/debug-helpers.js"></script>
)}
```

## 🔍 Диагностика проблем

### Проблемы с автоскролом

1. **Откройте консоль разработчика**
2. **Выполните:** `ZeroDolgDebug.checkFAQ()`
3. **Проверьте:**
   - Найден ли элемент с ID "faq"
   - Найдены ли ссылки с href="#faq"
   - Правильно ли определяется высота заголовка
   - Нет ли конфликтов между обработчиками

4. **Тест прокрутки:** `ZeroDolgDebug.testScroll('faq')`

### Проблемы с баннером

1. **Откройте консоль разработчика**
2. **Выполните:** `ZeroDolgDebug.checkBanner()`
3. **Проверьте:**
   - Найден ли элемент баннера
   - Какие CSS классы применены
   - Состояние sessionStorage
   - Позиционирование и z-index

4. **Сброс состояния:** `ZeroDolgDebug.resetBanner()` + обновить страницу

## 🐛 Частые проблемы и решения

### FAQ не прокручивается

1. Проверьте, что элемент имеет ID "faq"
2. Убедитесь, что нет JS ошибок, блокирующих выполнение скриптов
3. Проверьте конфликты CSS, которые могут блокировать scroll-behavior

### Баннер не появляется

1. Проверьте консоль на ошибки JavaScript
2. Очистите sessionStorage: `sessionStorage.removeItem('offerBannerClosed')`
3. Убедитесь, что компонент SpecialOfferBanner подключен в layout
4. Проверьте CSS z-index конфликты

### Баннер появляется сразу

1. Проверьте логи в консоли - должна быть задержка 5 секунд
2. Убедитесь, что useState инициализируется с false для isVisible
