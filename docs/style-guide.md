# Руководство по стилю кода

## Общие принципы

### 1. Согласованность
- Следуйте существующим паттернам в кодовой базе
- Поддерживайте единый стиль во всем проекте
- Используйте одни и те же подходы для решения схожих задач

### 2. Читаемость
- Код должен быть самодокументируемым
- Используйте понятные имена переменных, функций и компонентов
- Разделяйте логические блоки пустыми строками
- Комментируйте сложную или неочевидную логику

### 3. Простота
- Предпочитайте простые решения сложным
- Избегайте преждевременной оптимизации
- Следуйте принципу KISS (Keep It Simple, Stupid)

### 4. Производительность
- Учитывайте производительность при написании кода
- Избегайте ненужных повторяющихся вычислений
- Оптимизируйте компоненты и рендеринг при необходимости

## Форматирование

### Отступы
- Используйте 2 пробела для отступов (не табы)
- Всегда используйте новую строку в конце файлов

### Длина строк
- Максимум 100 символов для кода
- Максимум 80 символов для комментариев

### Точка с запятой
- Используйте точки с запятой в JavaScript/TypeScript
- В TypeScript используйте точки с запятой для завершения операторов

### Кавычки
- Используйте одинарные кавычки для строк в JavaScript
- Используйте двойные кавычки для HTML атрибутов

### Замыкающие запятые
- Используйте замыкающие запятые для многострочных объектов/массивов

## Именование

### Файлы и папки
- Компоненты: PascalCase (например, `ReviewCard.astro`)
- Секции: PascalCase (например, `HeroSection.astro`)
- Страницы: kebab-case (например, `test-reviews.astro`)
- Стили: kebab-case с подчеркиванием (например, `_reviews.css`)
- Скрипты: kebab-case (например, `reviews-carousel.js`)
- Утилиты: kebab-case (например, `content-service.js`)

### Код
- Компоненты: PascalCase
- Функции: camelCase
- Переменные: camelCase
- Константы: UPPER_SNAKE_CASE
- CSS классы: BEM методология (block__element--modifier)
- CSS переменные: kebab-case с префиксом (например, `--color-primary`)

### TypeScript
- Интерфейсы: PascalCase с заглавным `I` (например, `IUser`)
- Типы: PascalCase (например, `ButtonType`)

## JavaScript/TypeScript

### Объявление переменных
- Используйте `const` по умолчанию
- Используйте `let` только если переменная будет изменяться
- Избегайте `var`

### Функции
- Используйте стрелочные функции для коротких функций
- Используйте объявления функций для именованных функций
- Используйте деструктуризацию при необходимости

### TypeScript
- Описывайте типы для всех пропсов компонентов
- Используйте строгую типизацию
- Избегайте `any`, когда это возможно

## Компоненты

### Создание компонентов
- Определяйте интерфейс пропсов с TypeScript
- Используйте семантический HTML
- Добавляйте ARIA атрибуты для доступности
- Обеспечивайте адаптивный дизайн
- Используйте оптимизацию производительности (ленивая загрузка и т.д.)

### Размещение компонентов
- UI компоненты: `/src/components/ui/` (кнопки, карточки и т.д.)
- Формы: `/src/components/forms/` (элементы и макеты форм)
- Секции: `/src/components/sections/` (Hero, Benefits и т.д.)
- Макеты: `/src/components/layout/` или `/src/layouts/`
- Интерактивные компоненты: `/src/islands/` (только компоненты Preact)
- Фиче-специфичные компоненты: `/src/features/[feature-name]/ui/`
- Сложные виджеты: `/src/widgets/`

## Доступность (a11y)

### Семантический HTML
- Используйте теги по их назначению
- Применяйте заголовки в правильной иерархии
- Используйте элементы списка для списков

### ARIA атрибуты
- Добавляйте `role` атрибуты при необходимости
- Используйте `aria-label` и `aria-describedby`
- Применяйте `aria-hidden` для скрытых элементов

### Клавиатурная навигация
- Обеспечьте логический порядок табуляции
- Добавляйте визуальные индикаторы фокуса
- Обеспечьте доступность динамического контента
