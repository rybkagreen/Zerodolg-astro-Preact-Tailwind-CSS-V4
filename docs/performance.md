# Руководство по оптимизации производительности

## Общая информация

Это руководство описывает подходы и методы оптимизации производительности проекта ZeroDolg Astro для обеспечения быстрой загрузки и отзывчивости сайта.

## Метрики производительности

### Core Web Vitals
- **Largest Contentful Paint (LCP)**: < 2.5 секунды
- **First Input Delay (FID)**: < 100 миллисекунд
- **Cumulative Layout Shift (CLS)**: < 0.1

### Дополнительные метрики
- **Time to Interactive (TTI)**: < 5 секунд
- **First Contentful Paint (FCP)**: < 1.8 секунды
- **Speed Index**: < 3.4 секунды

## Оптимизация изображений

### Форматы изображений
- **WebP**: Основной формат для современных браузеров
- **AVIF**: Для поддерживаемых браузеров
- **JPEG/PNG**: Резервные форматы

### Размеры изображений
```html
<!-- Адаптивные изображения с srcset -->
<img 
  src="/images/photo-400w.webp" 
  srcset="/images/photo-400w.webp 400w,
          /images/photo-800w.webp 800w,
          /images/photo-1200w.webp 1200w"
  sizes="(max-width: 400px) 400px,
         (max-width: 800px) 800px,
         1200px"
  alt="Описание изображения"
  loading="lazy"
>
```

### Ленивая загрузка
```html
<!-- Использование loading="lazy" для изображений вне экрана -->
<img src="image.jpg" alt="Описание" loading="lazy">

<!-- Ленивая загрузка для iframe -->
<iframe src="video.html" loading="lazy"></iframe>
```

### Оптимизация изображений
- **Размер файла**: < 200KB для основных изображений
- **Сжатие**: Использование современных алгоритмов сжатия
- **Размеры**: Точные размеры для каждого контекста
- **Кэширование**: Правильные заголовки кэширования

## Оптимизация CSS

### Минификация
- Использование минифицированного CSS в продакшене
- Удаление неиспользуемого CSS
- Объединение файлов для уменьшения количества запросов

### Критический CSS
```html
<!-- Встраивание критического CSS в <head> -->
<style>
  /* Критические стили для Above-the-fold контента */
  .header { /* стили */ }
  .hero { /* стили */ }
  .main { /* стили */ }
</style>

<!-- Асинхронная загрузка остального CSS -->
<link rel="preload" href="/styles/main.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
```

### Оптимизация селекторов
- Минимизация вложенности (максимум 3 уровня)
- Использование эффективных селекторов
- Избегание универсальных селекторов (*)
- Использование BEM для предсказуемой специфичности

## Оптимизация загрузки страниц

### Preload и Prefetch
```html

<link rel="preload" href="/styles/critical.css" as="style">
<link rel="preload" href="/fonts/inter-regular.woff2" as="font" type="font/woff2" crossorigin>

```

### Resource Hints
```html

<link rel="dns-prefetch" href="//cms.alibabacloud.com">

<!-- Preconnect для критических ресурсов -->

```

## Оптимизация контента из CMS

### Кэширование данных
```javascript
// src/lib/content-service.js
const CACHE_TTL = 5 * 60 * 1000; // 5 минут
const cache = new Map();

export async function getCachedTeamMembers() {
  const cacheKey = 'team-members';
  const cached = cache.get(cacheKey);
  
  if (cached && Date.now() - cached.timestamp < CACHE_TTL) {
    return cached.data;
  }
  
  const data = await getTeamMembers();
  cache.set(cacheKey, {
    data,
    timestamp: Date.now()
  });
  
  return data;
}
```

- Объединение нескольких запросов в один
- Использование пагинации для больших коллекций
- Фильтрация данных на стороне сервера

## Проблемы и решения

### Частые проблемы производительности

#### Медленная загрузка изображений
**Решение**:
- Использовать WebP формат
- Реализовать ленивую загрузку
- Оптимизировать размеры файлов
- Использовать srcset для адаптивных изображений

#### Сдвиги макета (CLS)
**Решение**:
- Указывать размеры изображений
- Резервировать место для динамического контента
- Избегать вставки шрифтов без font-display: swap
- Тестировать на разных устройствах

#### Медленное первое взаимодействие (FID)
**Решение**:
- Минимизировать JavaScript бандл
- Использовать code splitting
- Оптимизировать обработчики событий
- Уменьшить количество сторонних скриптов
