## Обзор

Сайт zerodolg.ru работает с октября 2025. Работа делится на 3 потока:
техдолг из аудита (WS-1) · премиум-редизайн главной (WS-2) · поток корректировок (WS-3).
Каждый крупный кусок = один декларативный промт под агента (цель + DoD + acceptance-контракт),
STOP на границе, fresh session per крупный блок.

## WS-1 — Техдолг из аудита `[ ]`

- **Цель:** закрыть конкретные дефекты, найденные при обходе сайта (лист «Найденные правки»).
- **Scope (по приоритету):**
  1. **[Высокий]** Починить сломанную статью `/blog/blog-bankrotstvo-pensionera` (не парсится
     frontmatter, заголовок «Без названия»).
  2. **[Высокий]** Актуализировать `sitemap.xml` (+7 отсутствующих URL) и убрать/починить осиротевший URL.
  3. **[Высокий]** Ручная проверка мобильной адаптивности + фиксы (аудит визуально не подтвердил).
  4. **[Средний]** Интегрировать `/privacy/`, `/terms/` в общий шаблон (шапка/футер/брендинг).
  5. **[Низкий]** Убрать дублирование виджета обратного звонка.
  6. **[Опц.]** Унифицировать ~12 форм/модалок в один конфигурируемый компонент (окупается на будущих правках).
- **DoD:** статья рендерится корректно; sitemap валиден и полон; мобильный вид проверен и без поломок;
  `npm run type-check && npm run lint && npm run build` зелёные (⚠️ без `&& npm run test` — скрипта
  нет, см. `ARCHITECTURE.md §1`); PROTECTED-дифф контролируем.
- **Смета:** 66–159 тыс ₽ (без п.6 — 42–99 тыс).
- **STOP:** после каждого высокоприоритетного фикса (осмысленная граница) или связкой п.1+п.2.

## WS-2 — Премиум-редизайн главной `[ ]`

- **Цель:** переработать главную по дизайн-брифу — от давления к спокойной экспертизе.
- **Источник:** `Дизайн-бриф_ZeroDolg_главная.html` (палитра, типо-шкала, отступы, карта секций, a11y;
  артефакт у владельца, не трекается в этом репозитории).
- **Этапы (порядок по влиянию на доверие):**
  - **P1 · Быстрые победы** — убрать фейковые триггеры (счётчик «онлайн», сломанный таймер,
    зацикленный тикер), keyword-stuffing; фикс загрузки фото эксперта; Федресурс как доказательство.
  - **P2 · Hero и тон** — переписать копирайт (сдержанно, без «гарантированно»), новая композиция,
    одна форма с минимумом полей.
  - **P3 · Дизайн-система** — токены палитры (тил-акцент), типо-шкала, отступы; применить по блокам.
  - **P4 · Структура** — сократить до 6–7 секций, объединить дубли, один целевой CTA, отзывы сеткой.
  - **P5 · Доступность и семантика** — лендмарки, alt/aria, focus-state, WCAG-контраст, удаление SEO-мусора.
- **DoD:** каждый этап — визуально проверен (desktop + mobile), a11y не регрессировала, CSP цела,
  real CI (`type-check`+`lint`+`build`) зелёный.
- **Смета:** приоритетный старт (P1+P2) — 123–258 тыс ₽; полный — 447–930 тыс ₽; без дизайн-макетов
  (если дизайн свой) — 357–750 тыс ₽.
- **STOP:** на границе каждого этапа (P1…P5) + ревью владельца перед выкаткой на прод.
- **Blocker:** согласование направления/макета заказчиком.

## Conventions

- Промт = поток/этап целиком. Декларативно + DoD + acceptance-контракт.
- STOP на границах этапов. Fresh session per крупный блок.
- 4 принципа + границы FSD. Git policy (feature → PR → `master`; деплой из `master`).
- Production changes → gate-checks (`type-check`+`lint`+`build`; `+test` только когда раннер
  реально появится в репозитории); gate упал → STOP.
- PII/security-чувствительное (формы/Bitrix/аналитика) — узкими отдельными промтами.
