# Puppeteer MCP Server - Руководство пользователя

## Описание

Puppeteer MCP Server предоставляет возможность автоматизировать браузерные операции через протокол MCP (Model Context Protocol). Этот сервер позволяет программно управлять браузером Chrome/Chromium для навигации по веб-страницам, создания скриншотов, извлечения контента и взаимодействия с элементами страницы.

## Установка и настройка

### Требования

- Node.js 18.17.1 или выше
- Puppeteer (уже установлен в проекте)
- MCP SDK (уже установлен)

### Запуск сервера

```bash
# Запуск через npm скрипт
npm run mcp:puppeteer

# Или напрямую
npm run mcp:server
```

## Доступные инструменты

### 1. `navigate_to_url`

Навигация к указанному URL.

**Параметры:**

- `url` (string, обязательный) - URL для перехода
- `waitUntil` (string, опциональный) - Условие ожидания загрузки: `"load"`, `"domcontentloaded"`, `"networkidle0"`, `"networkidle2"` (по умолчанию)

**Пример:**

```json
{
  "name": "navigate_to_url",
  "arguments": {
    "url": "https://example.com",
    "waitUntil": "networkidle2"
  }
}
```

### 2. `take_screenshot`

Создание скриншота текущей страницы.

**Параметры:**

- `name` (string, опциональный) - Имя файла скриншота (по умолчанию: "screenshot")
- `fullPage` (boolean, опциональный) - Захват всей страницы (по умолчанию: true)
- `element` (string, опциональный) - CSS селектор конкретного элемента

**Пример:**

```json
{
  "name": "take_screenshot",
  "arguments": {
    "name": "homepage",
    "fullPage": true
  }
}
```

### 3. `get_page_content`

Получение HTML содержимого страницы.

**Параметры:**

- `selector` (string, опциональный) - CSS селектор для получения контента конкретного элемента

**Пример:**

```json
{
  "name": "get_page_content",
  "arguments": {
    "selector": ".main-content"
  }
}
```

### 4. `get_page_title`

Получение заголовка текущей страницы.

**Параметры:** нет

**Пример:**

```json
{
  "name": "get_page_title",
  "arguments": {}
}
```

### 5. `get_page_url`

Получение текущего URL страницы.

**Параметры:** нет

### 6. `click_element`

Клик по элементу на странице.

**Параметры:**

- `selector` (string, обязательный) - CSS селектор элемента

**Пример:**

```json
{
  "name": "click_element",
  "arguments": {
    "selector": "#submit-button"
  }
}
```

### 7. `type_text`

Ввод текста в поле ввода.

**Параметры:**

- `selector` (string, обязательный) - CSS селектор поля ввода
- `text` (string, обязательный) - Текст для ввода

**Пример:**

```json
{
  "name": "type_text",
  "arguments": {
    "selector": "#search-input",
    "text": "Hello, World!"
  }
}
```

### 8. `wait_for_element`

Ожидание появления элемента на странице.

**Параметры:**

- `selector` (string, обязательный) - CSS селектор элемента
- `timeout` (number, опциональный) - Таймаут в миллисекундах (по умолчанию: 10000)

### 9. `evaluate_javascript`

Выполнение JavaScript кода в контексте страницы.

**Параметры:**

- `code` (string, обязательный) - JavaScript код для выполнения

**Пример:**

```json
{
  "name": "evaluate_javascript",
  "arguments": {
    "code": "document.title"
  }
}
```

### 10. `get_viewport_size`

Получение размеров области просмотра.

**Параметры:** нет

### 11. `set_viewport_size`

Установка размеров области просмотра.

**Параметры:**

- `width` (number, обязательный) - Ширина в пикселях
- `height` (number, обязательный) - Высота в пикселях

## Конфигурация

### VS Code

Конфигурация находится в файле `.vscode/mcp.json`:

```json
{
  "mcpServers": {
    "puppeteer-browser": {
      "command": "node",
      "args": ["mcp-puppeteer-server.js"],
      "env": {}
    }
  },
  "servers": {
    "puppeteer": {
      "type": "stdio",
      "command": "node",
      "args": ["mcp-puppeteer-server.js"]
    }
  }
}
```

### Qwen AI

Конфигурация находится в файле `.qwen/mcp.json`:

```json
{
  "mcpServers": {
    "puppeteer-browser": {
      "command": "node",
      "args": ["mcp-puppeteer-server.js"],
      "env": {}
    }
  }
}
```

## Использование

### Базовый сценарий

1. **Запустите MCP сервер:**

   ```bash
   npm run mcp:puppeteer
   ```

2. **Подключите MCP клиент** (например, Claude Desktop, VS Code с MCP расширением, или Qwen)

3. **Используйте доступные инструменты:**
   - Перейдите на веб-страницу с помощью `navigate_to_url`
   - Сделайте скриншот с помощью `take_screenshot`
   - Извлеките контент с помощью `get_page_content`
   - Взаимодействуйте с элементами используя `click_element` и `type_text`

### Пример сценария

```bash
# 1. Запустить сервер
npm run mcp:puppeteer

# 2. В MCP клиенте выполнить последовательность команд:
# - Перейти на сайт
# - Сделать скриншот
# - Получить заголовок
# - Найти и нажать кнопку
```

## Скриншоты

Все скриншоты сохраняются в папку `screenshots/` с автоматически генерируемыми именами включающими временные метки.

## Устранение неполадок

### Проблемы с браузером

Если браузер не запускается, убедитесь что:

- У вас достаточно системных ресурсов
- Нет конфликтующих процессов Chrome
- Права доступа позволяют запуск браузера

### Проблемы с сетью

При навигации к внешним сайтам проверьте:

- Интернет соединение
- Настройки брандмауэра
- Корректность URL

### Логи и отладка

Сервер выводит отладочную информацию в stderr. Для детальной отладки можно изменить уровень логирования в коде сервера.

## Ограничения

- Сервер работает в headless режиме (без графического интерфейса)
- Одновременно может работать только один экземпляр браузера
- Размер скриншотов ограничен доступной памятью
- Некоторые сайты могут блокировать автоматизированные браузеры

## Безопасность

- Сервер выполняет код JavaScript в контексте браузера - будьте осторожны с недоверенным кодом
- При навигации к внешним сайтам соблюдайте их условия использования
- Скриншоты могут содержать конфиденциальную информацию
