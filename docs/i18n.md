# Руководство по интернационализации (i18n)

## Общая информация

Это руководство описывает подход к интернационализации проекта ZeroDolg Astro, включая структуру локализации, инструменты и процессы.

## Поддерживаемые языки

### Основной язык
- **Русский (ru-RU)** - основной язык контента

### Дополнительные языки
- **Английский (en-US)** - для технической документации и интерфейса
- **Испанский (es-ES)** - планируется к добавлению
- **Немецкий (de-DE)** - планируется к добавлению

## Интеграция с Astro

### Конфигурация i18n
```javascript
// src/i18n/index.js
import ruCommon from './ru/common.json';
import enCommon from './en/common.json';
// ... другие импорты

export const translations = {
  'ru-RU': {
    common: ruCommon,
    navigation: ruNavigation,
    // ... другие разделы
  },
  'en-US': {
    common: enCommon,
    navigation: enNavigation,
    // ... другие разделы
  }
};

export const defaultLocale = 'ru-RU';
export const supportedLocales = ['ru-RU', 'en-US'];

export function getTranslation(locale, section, key) {
  const translation = translations[locale]?.[section]?.[key];
  if (translation) return translation;
  
  // Fallback к дефолтной локали
  return translations[defaultLocale]?.[section]?.[key] || key;
}
```

### Использование в компонентах
```astro
---
// src/components/sections/Hero.astro
import { getTranslation } from '../../i18n/index.js';

const t = (section, key) => getTranslation(Astro.locals.locale || 'ru-RU', section, key);
---

<section class="hero">
  <h1>{t('common', 'site.title')}</h1>
  <p>{t('hero', 'description')}</p>
  <button>{t('common', 'actions.consultation')}</button>
</section>
```

### Динамический контент (CMS)
Для контента из Alibaba CMS реализована поддержка мультиязычности:

```javascript
// src/lib/content-service.js
export async function getLocalizedTeamMembers(locale = 'ru-RU') {
  try {
    // Запрос к CMS с указанием локали
    const response = await fetchFromCMS(`/api/v1/content/team?locale=${locale}`);
    return response.data || [];
  } catch (error) {
    console.error(`[i18n] Error fetching localized team members for ${locale}:`, error);
    return [];
  }
}
```

## Управление переводами

### Инструменты перевода
- **Crowdin** - для управления переводами сообществом
- **JSON редакторы** - для ручного редактирования
- **Скрипты синхронизации** - для автоматизации процессов

### Процесс добавления нового языка
1. Создать директорию для нового языка в `src/i18n/`
2. Скопировать структуру файлов из существующего языка
3. Перевести все строки
4. Добавить язык в `supportedLocales`
5. Создать маршруты для нового языка
6. Обновить селектор языка
7. Протестировать все страницы

### Скрипты синхронизации
```javascript
// scripts/sync-translations.js
import fs from 'fs-extra';
import { supportedLocales, defaultLocale } from '../src/i18n/index.js';

async function syncTranslations() {
  const defaultTranslations = await fs.readJSON(`src/i18n/${defaultLocale}/common.json`);
  
  for (const locale of supportedLocales) {
    if (locale === defaultLocale) continue;
    
    const localePath = `src/i18n/${locale}/common.json`;
    const existingTranslations = await fs.readJSON(localePath);
    
    // Синхронизация ключей
    const syncedTranslations = {
      ...defaultTranslations,
      ...existingTranslations
    };
    
    await fs.writeJSON(localePath, syncedTranslations, { spaces: 2 });
  }
}

syncTranslations();
```

## Переключение языков

### Компонент выбора языка
```astro
---
// src/components/ui/LanguageSwitcher.astro
import { supportedLocales, defaultLocale } from '../../i18n/index.js';

interface Props {
  currentLocale: string;
}

const { currentLocale } = Astro.props;

const getLocaleName = (locale) => {
  const names = {
    'ru-RU': 'Русский',
    'en-US': 'English'
  };
  return names[locale] || locale;
};

const getLocaleUrl = (locale) => {
  if (locale === defaultLocale) return '/';
  return `/${locale.split('-')[0]}/`;
};
---

<div class="language-switcher">
  <select 
    onchange={`window.location.href = this.value`}
    value={getLocaleUrl(currentLocale)}
  >
    {supportedLocales.map(locale => (
      <option value={getLocaleUrl(locale)}>
        {getLocaleName(locale)}
      </option>
    ))}
  </select>
</div>
```

## Тестирование локализации

### Чек-лист тестирования
- [ ] Отображение правильного языка по умолчанию
- [ ] Корректная смена языка
- [ ] Все строки переведены
- [ ] Нет сломанных ссылок между языками
- [ ] Правильные мета-теги для каждого языка
- [ ] Адаптация форматов дат и чисел
- [ ] Правильная маршрутизация

### Автоматическое тестирование
```javascript
// tests/i18n.test.js
import { test, expect } from '@playwright/test';

test.describe('Internationalization', () => {
  test('should display Russian content by default', async ({ page }) => {
    await page.goto('/');
    await expect(page.locator('h1')).toContainText('Банкротство физических лиц');
  });

  test('should switch to English', async ({ page }) => {
    await page.goto('/');
    await page.selectOption('.language-switcher select', '/en/');
    await expect(page.locator('h1')).toContainText('Personal Bankruptcy');
  });
});
```

## Best Practices

### Работа со строками
- [ ] Использовать описательные ключи (например, `reviews.title` вместо `title`)
- [ ] Группировать связанные строки в объекты
- [ ] Добавлять комментарии к сложным строкам
- [ ] Избегать жестко закодированных строк в компонентах

### Форматирование
- [ ] Использовать параметризованные строки для динамических значений
- [ ] Учитывать порядок слов в разных языках
- [ ] Правильно обрабатывать множественные формы
- [ ] Учитывать различия в регистрах

### Производительность
- [ ] Кэшировать переводы
- [ ] Минимизировать количество HTTP запросов
- [ ] Использовать ленивую загрузку для неиспользуемых языков
- [ ] Оптимизировать размер файлов локализации
