# Компоненты проекта

## Общая информация

Компоненты в проекте ZeroDolg Astro разделены на три основные категории:
1. **UI компоненты** - переиспользуемые элементы интерфейса
2. **Секции** - крупные блоки страниц
3. **Интерактивные компоненты** - компоненты с JavaScript функциональностью

## UI Компоненты

### Button
**Расположение**: `/src/components/ui/Button.astro`

Базовый компонент кнопки с различными вариантами оформления.

**Props**:
- `variant`: 'primary' | 'secondary' | 'outline' | 'ghost' (по умолчанию 'primary')
- `size`: 'sm' | 'md' | 'lg' (по умолчанию 'md')
- `disabled`: boolean (по умолчанию false)
- `type`: 'button' | 'submit' | 'reset' (по умолчанию 'button')

**Пример использования**:
```astro
<Button variant="primary" size="lg">Отправить</Button>
```

### Card
**Расположение**: `/src/components/ui/Card.astro`

Карточка для отображения контента в выделенном блоке.

**Props**:
- `title`: string (заголовок карточки)
- `description`: string (описание)
- `image`: string (путь к изображению)

**Пример использования**:
```astro
<Card 
  title="Услуга" 
  description="Описание услуги" 
  image="/images/service.jpg"
/>
```

### Modal
**Расположение**: `/src/components/ui/Modal.astro`

Модальное окно для отображения контента поверх основной страницы.

**Props**:
- `isOpen`: boolean (открыто ли модальное окно)
- `title`: string (заголовок модального окна)
- `onClose`: function (функция закрытия)

**Пример использования**:
```astro
<Modal isOpen={isModalOpen} title="Заголовок" onClose={closeModal}>
  <!-- Содержимое модального окна -->
</Modal>
```

## Создание новых компонентов

### Шаблон компонента
```astro
---
/**
 * @component ComponentName
 * @description Краткое описание компонента
 * @example
 * <ComponentName prop="value" />
 */

interface Props {
  // Определение props с TypeScript
}

const { } = Astro.props;
---

<div class="component-name">
  <!-- Семантическая HTML разметка -->
</div>

<style>
  /* CSS стили по методологии BEM */
  .component-name {
    /* Стили */
  }
</style>
```

### Рекомендации по созданию компонентов
1. Использовать TypeScript для типизации props
2. Следовать методологии BEM для именования классов
3. Использовать семантические HTML элементы
4. Добавлять ARIA атрибуты для доступности
5. Обеспечивать адаптивность
6. Добавлять документацию в начале файла
7. Использовать CSS custom properties для темизации
8. Минимизировать JavaScript функциональность
