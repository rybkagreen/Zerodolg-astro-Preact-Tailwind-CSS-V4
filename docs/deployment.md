# Руководство по развертыванию

## Обзор

Документ описывает процесс развертывания приложения ZeroDolg Astro на продакшен-сервере. Проект использует Astro v5.13.5 и развертывается как статический сайт.

## Подготовка к развертыванию

### Требования

- Node.js >= 18.17.1
- npm >= 8.0.0
- Установленный Git

### Переменные окружения

Перед развертыванием убедитесь, что настроены все необходимые переменные окружения:

```env
# Alibaba CMS
CMS_API_BASE=https://your-cms-endpoint.alibabacloud.com
CMS_API_KEY=your-api-key-here

# Аналитика (если используется)
GA_MEASUREMENT_ID=your-google-analytics-id
YANDEX_METRICA_ID=your-yandex-metrica-id

# Дополнительные переменные для продакшена
NODE_ENV=production
```

### Проверка перед развертыванием

1. Убедитесь, что все тесты проходят:
   ```
   npm run test
   ```

2. Проверьте типизацию:
   ```
   npm run type-check
   ```

3. Проверьте линтинг:
   ```
   npm run lint
   ```

4. Выполните сборку проекта:
   ```
   npm run build
   ```

## Локальная сборка

### Сборка для продакшена

```
npm run build
```

Эта команда:
- Очищает старые файлы
- Проверяет типы TypeScript
- Собирает проект для продакшена
- Создает статические файлы в директории `dist/`

### Сборка с оптимизациями

```
npm run build:prod
```

Эта команда включает дополнительные оптимизации для продакшена.

### Предварительный просмотр

```
npm run preview
```

Запускает локальный сервер для предварительного просмотра собранного проекта.

## Развертывание

### На общедоступный хостинг

Проект можно развернуть на любом хостинге, который поддерживает статические файлы (Netlify, Vercel, GitHub Pages, и т.п.).

1. Соберите проект командой `npm run build`
2. Загрузите содержимое директории `dist/` на ваш хостинг
3. Убедитесь, что сервер настроен для обслуживания файла `index.html` по всем маршрутам

### На собственный сервер

#### Apache

1. Убедитесь, что модуль `mod_rewrite` включен
2. Создайте файл `.htaccess` в корне проекта:
```apache
Options -MultiViews
RewriteEngine On

RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^(.*)$ index.html [QSA,L]
```

#### Nginx

Настройте сервер в Nginx следующим образом:

```nginx
server {
  listen 80;
  server_name your-domain.com;
  root /path/to/your/project/dist;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  # Разрешить доступ к статическим файлам
  location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }
}
```

### Автоматическое развертывание

#### GitHub Actions

В проекте настроена CI/CD pipeline через GitHub Actions. Рабочий процесс находится в `.github/workflows/ci.yml`.

Для настройки автоматического развертывания:

1. Добавьте необходимые секреты в репозиторий GitHub:
   - `CMS_API_KEY`: API ключ для CMS
   - `DEPLOY_HOST`: Хост для развертывания
   - `DEPLOY_USER`: Пользователь для SSH
   - `DEPLOY_KEY`: Приватный SSH ключ
   - `DEPLOY_PATH`: Путь к директории на сервере

2. Обновите файл `.github/workflows/ci.yml` под ваши требования развертывания

#### Скрипты развертывания

В проекте есть скрипты для развертывания в директории `scripts/deploy/`:

- `deploy.sh` - основной скрипт развертывания
- `create-backup.js` - создание резервной копии
- `rollback.js` - откат к предыдущей версии
- `post-build-verification.js` - пост-проверка после сборки

## Мониторинг и поддержка

### Проверка после развертывания

После развертывания выполните проверки:

1. Проверьте доступность сайта
2. Протестируйте основные функции (формы, навигация, интерактивные компоненты)
3. Проверьте отсутствие ошибок в консоли браузера
4. Проверьте работу аналитики
5. Убедитесь, что все страницы индексируются поисковыми системами

### Логирование

Все скрипты развертывания ведут логи в директории `logs/`. Убедитесь, что у вас есть доступ к этим логам для диагностики проблем.

### Резервное копирование

Перед каждым обновлением создается резервная копия текущей версии. Скрипт резервного копирования: `scripts/deploy/create-backup.js`.

### Откат

В случае проблем с новой версией можно выполнить откат к предыдущей стабильной версии с помощью скрипта: `scripts/deploy/rollback.js`.

## Оптимизация после развертывания

### Кеширование

Настройте кеширование статических файлов:

- HTML файлы: не кешировать или кешировать на короткое время
- JS/CSS файлы: кешировать на длительное время (год), т.к. они имеют хэши в именах
- Изображения и шрифты: кешировать на длительное время

### Безопасность

Убедитесь, что сервер настроен правильно:

- Используется HTTPS
- Установлены заголовки безопасности (CSP, X-Frame-Options и т.д.)
- Закрыт доступ к служебным файлам (.git, конфиги и т.д.)

## Тестирование продакшен-окружения

### Lighthouse аудит

Выполните аудит производительности с помощью скрипта:
```
npm run maintenance:lighthouse
```

### Тестирование форм

Проверьте работу всех форм на продакшене, включая валидацию и отправку данных.

### Проверка SEO

Убедитесь, что:
- Все страницы корректно индексируются
- META-теги и структурированные данные правильно отображаются
- Sitemap.xml доступен и содержит все важные страницы
- robots.txt разрешает поисковым системам сканирование

## Устранение неполадок

### Возможные проблемы

1. **Страницы не находятся (404)**: Проверьте настройки сервера на предмет обслуживания всех маршрутов через `index.html`

2. **Стили не загружаются**: Проверьте, что все CSS файлы доступны и правильно подключены

3. **Интерактивные компоненты не работают**: Проверьте, что JS файлы загружаются и не блокируются политикой безопасности

4. **CMS контент не загружается**: Проверьте настройки CORS и переменные окружения

### Отладка

Для отладки продакшен-версии:
- Проверьте консоль браузера на наличие ошибок
- Убедитесь, что все ресурсы (CSS, JS, изображения) загружаются без ошибок
- Используйте инструменты разработчика для проверки сети и производительности
