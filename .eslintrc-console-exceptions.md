# Console Exceptions для ESLint

Этот файл документирует легитимное использование console statements в проекте.

## Файлы с разрешенным использованием console:

### Logger Implementation

- `src/shared/lib/logger.ts` - **Logger реализация** (обязательно использует
  console)
- `src/shared/lib/performance.ts` - **Performance tracking** (для мониторинга)
- `src/shared/hooks/usePerformanceMonitor.ts` - **Performance monitoring hook**

### Debug & Development

- `src/features/analytics/analytics.ts` - Debug logging для аналитики
- `src/shared/lib/analytics.ts` - Analytics logging
- `src/shared/analytics/tracking-config.ts` - Development logging

### Error Handling

- `src/islands/forms/FormEnhancedFinal.tsx` - Error boundary logging
- `src/shared/hooks/useLocalStorage.ts` - Error logging для localStorage
- `src/shared/lib/hash-utils.ts` - Crypto fallback warnings
- `src/shared/lib/bitrix-callback.ts` - Error logging
- `src/shared/lib/bitrix-callback-widget.ts` - Error logging
- `src/shared/lib/puppeteer-helper.ts` - Puppeteer initialization errors

### Analytics & Tracking

- `src/islands/sections/ProblemsInteractive.tsx` - Analytics logging
- `src/islands/sections/ReviewsEnhanced.tsx` - Analytics logging
- `src/islands/shared/interactive/lead-magnets.tsx` - Debug logging
- `src/islands/forms/form-logic.tsx` - Debug logging

## Рекомендации:

Все console statements должны иметь eslint-disable комментарий с пояснением:

```typescript
// eslint-disable-next-line no-console -- Reason for using console
console.log('message');
```
