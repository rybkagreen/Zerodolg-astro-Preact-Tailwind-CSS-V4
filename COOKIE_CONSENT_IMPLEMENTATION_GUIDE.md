## 📦 Что было создано

### 1. Новые файлы:

#### `src/shared/lib/consent-manager.ts`

- **Singleton класс** для управления согласием пользователя
- Реализует **Consent Mode v2** для Google Analytics
- Методы:
  - `hasAnalyticsConsent()` - проверка согласия
  - `acceptAll()` - принять все
  - `declineAll()` - отклонить все
  - `revokeConsent()` - отозвать согласие
- Хранит состояние в `localStorage`
- Срок действия: 365 дней

#### `src/shared/ui/CookieBanner.tsx`

- **Preact компонент** с Cookie баннером
- Responsive дизайн (mobile-first)
- Расположение: снизу экрана
- Анимации: slide-in, fade-in
- Два режима: компактный и развернутый
- Кнопки: "Принять все" и "Отклонить"

## 🚀 Как протестировать

### 1. Запустите проект локально:

```bash
npm run dev
```

### 2. Откройте браузер:

```
http://localhost:4321
```

### 3. Что должно произойти:

1. ✅ Через **1 секунду** после загрузки появится Cookie Banner внизу экрана
2. ✅ Баннер имеет темный дизайн с белым текстом
3. ✅ Есть кнопки "Принять все" (синяя) и "Отклонить" (серая)
4. ✅ Есть кнопка "Подробнее" для раскрытия дополнительной информации

### 4. Проверьте функциональность:

#### Сценарий A: Принять согласие

1. Нажмите **"Принять все"**
2. Баннер должен исчезнуть
3. Откройте **DevTools → Console**
4. Вы должны увидеть:
   ```
   ✅ User consent granted: {analytics: "granted", advertising: "denied", timestamp: ...}
   ✅ Google Consent Mode updated: ...
   ```
5. Откройте **DevTools → Application → Local Storage**
6. Должен быть ключ `user_consent_v2` со значением:
   ```json
   {"analytics":"granted","advertising":"denied","timestamp":...}
   ```
7. **Перезагрузите страницу** - баннер не должен появиться снова

#### Сценарий B: Отклонить согласие

1. Очистите localStorage: DevTools → Application → Clear site data
2. Перезагрузите страницу
3. Нажмите **"Отклонить"**
4. Баннер должен исчезнуть
5. В Console:
   ```
   ❌ User consent denied: {analytics: "denied", advertising: "denied", timestamp: ...}
   ```
6. Заполните и отправьте форму обратной связи
7. В Console должно быть:
   ```
   [Analytics] Conversion tracking: no user consent for Enhanced Conversions
   ```
8. **Персональные данные НЕ должны отправляться** в GA4

#### Сценарий C: Истечение срока

1. В DevTools → Console выполните:
   ```javascript
   import { consentManager } from '/src/shared/lib/consent-manager';
   consentManager.clearConsent();
   ```
2. Перезагрузите страницу - баннер появится снова

## 🔍 Проверка Enhanced Conversions

### С согласием пользователя:

1. Примите согласие через Cookie Banner
2. Заполните форму обратной связи
3. Откройте **DevTools → Network**
4. Фильтр: `google-analytics.com/g/collect`
5. Отправьте форму
6. Найдите запрос с параметрами:
   - `&em=` - хешированный email
   - `&ph=` - хешированный телефон
   - `&fn=` - хешированное имя
7. ✅ Данные должны быть в формате SHA-256 (длинная строка hex)

### Без согласия:

1. Отклоните согласие
2. Заполните форму
3. В Network запрос **НЕ должен содержать** параметры `&em=`, `&ph=`, `&fn=`
4. ✅ Базовое событие отправляется, но без персональных данных

### Через 24-48 часов после внедрения:

   - Должны быть отмечены: Email, Phone, Name

## 🐛 Troubleshooting

### Баннер не появляется

**Причина:** Согласие уже сохранено в localStorage

**Решение:**

1. DevTools → Application → Local Storage
2. Удалите ключ `user_consent_v2`
3. Перезагрузите страницу

### Согласие не сохраняется

**Причина:** localStorage заблокирован или недоступен

**Решение:**

1. Проверьте настройки браузера (cookies и site data должны быть разрешены)
2. Проверьте режим "Инкогнито" - там localStorage может быть ограничен

### Enhanced Conversions не работают

**Причина 1:** Согласие отклонено

**Решение:** Примите согласие через баннер

**Причина 2:** WebCrypto API недоступен

**Решение:**

1. Убедитесь, что сайт работает по HTTPS
2. Проверьте `window.crypto.subtle` в Console
3. Используйте современный браузер (Chrome 60+, Firefox 52+, Safari 11+)

### GA4 не получает данные

**Причина:** AdBlock или privacy расширения

**Решение:**

1. Отключите AdBlock для вашего сайта
2. Проверьте в режиме инкогнито без расширений
3. Проверьте Network → заблокированные запросы
