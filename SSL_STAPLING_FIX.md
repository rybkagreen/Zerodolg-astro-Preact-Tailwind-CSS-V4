# Исправление SSL-ошибки OCSP в nginx

## Проблема

Появляется ошибка в логах nginx:

```
OCSP_basic_verify() failed (SSL: error:13800065:OCSP routines::certificate verify error:Verify error: self-signed certificate in certificate chain) while requesting certificate status
```

Это ошибка указывает на проблему с проверкой статуса сертификата через OCSP.
Сертификат `zerodolg.ru.crt` содержит самоподписанный сертификат в цепочке, что
мешает проверке статуса.

## Причина

Ошибка возникает из-за включенной опции `ssl_stapling_verify on` в конфигурации
nginx, которая требует строгой проверки OCSP-ответа. При проверке цепочки
сертификатов обнаруживается самоподписанный сертификат, что приводит к ошибке
проверки.

## Решение

В конфигурационный файл `nginx-ssr.conf` были внесены следующие изменения:

1. Добавлен параметр
   `ssl_trusted_certificate /etc/ssl/certs/ca-certificates.crt;` - указывает на
   промежуточный сертификат центра сертификации для проверки OCSP
2. Изменено значение `ssl_stapling_verify on` на `ssl_stapling_verify off` -
   отключает строгую проверку stapling'а, что позволяет избежать ошибок с
   самоподписанными сертификатами в цепочке

## Результат

- SSL-защита сайта сохранена
- OCSP stapling все еще включен для получения и предоставления статуса
  сертификата клиентам
- Ошибка проверки OCSP устранена
- Безопасность соединения сохранена на приемлемом уровне

## Альтернативные решения

Если в будущем будут доступны правильные промежуточные сертификаты от центра
сертификации, можно:

1. Заменить `ssl_stapling_verify off` обратно на `ssl_stapling_verify on`
2. Указать конкретный промежуточный сертификат в параметре
   `ssl_trusted_certificate`, например:
   ```
   ssl_trusted_certificate /etc/ssl/certs/intermediate-ca.crt;
   ```
