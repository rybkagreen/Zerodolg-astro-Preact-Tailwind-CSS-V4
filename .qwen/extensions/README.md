# Расширения Qwen Code для разработки Astro-проекта

Этот каталог содержит расширения Qwen Code, которые помогут в разработке проекта на Astro.

## Установка

Расширения автоматически загружаются Qwen Code при запуске из каталогов:

- `<workspace>/.qwen/extensions` (для конкретного проекта)
- `<home>/.qwen/extensions` (глобальные расширения)

## Доступные расширения

### astro-dev

Команды для разработки Astro-проекта:

- `/dev` - Запустить сервер разработки Astro
- `/build` - Собрать проект Astro для продакшена
- `/preview` - Предварительный просмотр сборки
- `/test` - Запустить тесты проекта
- `/typecheck` - Проверить типы TypeScript
- `/lint` - Запустить линтер

### git-helper

Команды для работы с Git:

- `/status` - Показать статус репозитория Git
- `/add` - Добавить все изменения в индекс Git
- `/commit` - Создать коммит с сообщением

### deps-manager

Команды для управления зависимостями:

- `/install` - Установить все зависимости проекта
- `/update` - Обновить зависимости проекта

### perf-check

Команды для проверки производительности:

- `/audit` - Запустить аудит производительности с помощью Lighthouse

## Использование

После перезапуска Qwen Code все команды будут доступны через слэш-команды.
Например, для запуска сервера разработки выполните:

```
/dev
```

Для просмотра всех доступных команд используйте:

```
/help
```

## Конфликты команд

Если у пользователя есть команды с такими же именами, как у расширений, то:

- Пользовательская команда выполняется по своему имени (например, `/dev`)
- Команда расширения доступна с префиксом (например, `/astro-dev.dev`)

## Создание собственных расширений

Для создания собственного расширения:

1. Создайте каталог в `.qwen/extensions/` с именем вашего расширения
2. Добавьте файл `qwen-extension.json` с конфигурацией
3. Создайте каталог `commands` для команд расширения
4. Добавьте TOML-файлы команд в каталог `commands`

Пример структуры расширения:

```
.qwen/extensions/my-extension/
├── qwen-extension.json
└── commands/
    └── my-command.toml
```
