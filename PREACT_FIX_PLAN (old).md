# План решения проблем с Preact компонентами в Astro

## Дата создания: 19.09.2025

## Краткое описание проблемы
Все Preact компоненты в проекте ZeroDolg Astro не работают корректно после сборки. Компоненты рендерятся как статический HTML, но JavaScript функциональность не инициализируется, что приводит к:
- Неработающим кнопкам вне форм
- Нерабочей навигации по якорным ссылкам
- Неактивным динамически создаваемым элементам
- Общему отсутствию интерактивности

## Анализ корневых причин

### 1. Проблема с режимом вывода (Output Mode)
- **Текущее состояние**: `output: 'static'`
- **Проблема**: В статическом режиме Astro при определенных настройках может не включать JavaScript для гидратации компонентов
- **Решение**: Оставить `output: 'static'` но правильно настроить гидратацию компонентов

### 2. Проблемы с гидратацией компонентов
- **Текущее состояние**: Компоненты используют `client:load` и `client:idle` директивы
- **Проблема**: JavaScript бандлы не генерируются или не загружаются правильно
- **Решение**: Проверка и оптимизация директив гидратации

### 3. Конфликты конфигурации Vite
- **Текущее состояние**: Агрессивная минификация с `drop_console: true`
- **Проблема**: Критический код может удаляться при сборке
- **Решение**: Настройка более щадящей минификации

## Решения из сообщества разработчиков

### Решение 1: Использование @astrojs/preact с правильной конфигурацией для static mode
```javascript
// astro.config.mjs
export default defineConfig({
  integrations: [
    preact({
      compat: true,
      include: ['**/preact/**', '**/components/**/*.tsx']
    })
  ],
  output: 'static',
  vite: {
    ssr: {
      noExternal: ['preact', '@preact/preset-vite']
    },
    optimizeDeps: {
      include: ['preact', 'preact/hooks', 'preact/compat']
    }
  }
});
```

### Решение 2: Использование client:only директивы для проблемных компонентов
```astro
---
// Для компонентов которые не работают с client:load
---
<MyPreactComponent client:only="preact" />
```
