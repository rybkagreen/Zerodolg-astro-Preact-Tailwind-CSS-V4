# Отчет о проблемах с разметкой и их решении

## Описание проблем

1. Секция Problems перекрывала компоненты секции Solution (акварельный паттерн)
2. К нижней границе секции Solution нужно было применить маску с размытием для
   более мягкой границы с Stats
3. Секция Benefits прилипала к границе секции Calculator
4. Нижняя граница секции Benefits перекрывала компоненты секции Process
5. Нижняя граница секции Team перекрывала компоненты секции Конфиденциальность
6. Нижняя граница секции Конфиденциальность перекрывала компоненты секции FAQ
7. Нижняя граница секции FAQ перекрывала компоненты секции LeadMagnets

## Примененные решения

1. **Проблема с перекрытием Problems и Solution**: Уменьшен z-index секции
   Problems с 1 до 0, а также уменьшен нижний отступ с pb-32/40/48 до
   pb-16/24/32 для уменьшения перекрытия.

2. **Размытая граница Solution-Stats**: Добавлен более плавный градиент маски и
   фильтр размытия (blur(2px)) к фоновому изображению секции Solution для
   создания более мягкой границы с секцией Stats.

3. **Слипание Benefits и Calculator**: Увеличены отступы в Calculator с
   `pt-16 md:pt-24 lg:pt-32` до `pt-24 md:pt-32 lg:pt-40` для создания более
   заметного разделения между секциями.

4. **Перекрытие Benefits и Process**: Удалены отрицательные верхние отступы
   (-mt-24 md:-mt-32 lg:-mt-40) из секции Process, чтобы избежать перекрытия с
   секцией Benefits.

5. **Перекрытие других секций**: Удалены отрицательные отступы из других секций,
   где они вызывали подобные проблемы с перекрытием.

## Результат

Все перечисленные проблемы с разметкой были успешно исправлены. Построение
проекта проходит без ошибок, и визуальные элементы теперь правильно отображаются
без нежелательных перекрытий и с более плавными переходами между секциями.
