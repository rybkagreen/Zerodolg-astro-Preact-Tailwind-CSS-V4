# Рекомендации по настройке SSL для избежания ошибок OCSP

## Общая информация

OCSP (Online Certificate Status Protocol) используется для проверки статуса
сертификата в реальном времени. Ошибки OCSP могут возникать при проблемах с
цепочкой сертификатов или отсутствии необходимых промежуточных сертификатов.

## Текущее решение

В текущей конфигурации nginx мы отключили строгую проверку OCSP stapling с
помощью установки `ssl_stapling_verify off`. Это устраняет ошибку, но снижает
уровень проверки достоверности сертификата.

## Рекомендации по улучшению безопасности

### 1. Получение правильной цепочки сертификатов

Для восстановления строгой проверки OCSP рекомендуется:

1. Получить полную цепочку сертификатов от центра сертификации (CA)
2. Убедиться, что файл сертификата содержит не только ваш сертификат, но и все
   промежуточные сертификаты
3. Использовать команду для проверки целостности цепочки:
   ```
   openssl verify -CAfile ca-certificates.crt -untrusted intermediate.crt zerodolg.ru.crt
   ```

### 2. Правильная установка сертификатов

При получении нового сертификата:

1. Сохранить промежуточный сертификат в отдельный файл (например,
   `intermediate.crt`)
2. Объединить ваш сертификат и промежуточный сертификат в один файл в следующем
   порядке:
   ```
   -----BEGIN CERTIFICATE-----
   (Сертификат вашего домена)
   -----END CERTIFICATE-----
   -----BEGIN CERTIFICATE-----
   (Промежуточный сертификат)
   -----END CERTIFICATE-----
   ```
3. Использовать этот объединенный файл как `ssl_certificate`

### 3. Обновление конфигурации nginx при наличии правильных сертификатов

После получения правильной цепочки сертификатов можно вернуть строгую проверку:

```
ssl_stapling on;
ssl_stapling_verify on;  # Можно вернуть on при наличии правильных сертификатов
ssl_trusted_certificate /path/to/ca-certificates.crt;
```

### 4. Тестирование конфигурации

После любых изменений SSL рекомендуется протестировать конфигурацию:

1. Проверить синтаксис конфигурации:

   ```
   nginx -t
   ```

2. Проверить SSL конфигурацию с помощью внешних инструментов:
   - ssllabs.com
   - openssl s_client -connect zerodolg.ru:443 -servername zerodolg.ru

3. Проверить OCSP ответ:
   ```
   openssl s_client -connect zerodolg.ru:443 -servername zerodolg.ru -status
   ```

## Заключение

Текущее решение с отключением `ssl_stapling_verify` является приемлемым
временным решением для обеспечения работы сайта. В долгосрочной перспективе
рекомендуется получить правильную цепочку сертификатов от центра сертификации и
восстановить строгую проверку OCSP.
